## docker
1. Docker 对 NestJS+MySQL+Redis 后端项目的核心意义
环境统一：打包 NestJS 代码、MySQL/Redis 依赖及配置成镜像，解决 “开发机能跑、服务器跑崩”（比如本地 MySQL 8.0 vs 服务器 5.7 兼容问题）；
一键部署：无需在服务器手动装 Node.js/MySQL/Redis，执行 docker compose up 就能启动整套服务，省去环境配置耗时；
隔离稳定：MySQL/Redis 容器与宿主机 / 其他服务隔离，版本、端口、数据互不干扰，避免端口冲突 / 依赖污染；
扩缩容便捷：可快速复制容器实例（比如多开 Redis 节点），适配项目流量变化；
运维简化：通过镜像版本管理，回滚 / 升级服务只需替换镜像，无需手动改配置。
2. Docker 核心定义（Short 版）
Docker 是容器化工具，将应用 + 依赖打包成 “镜像”（只读模板），运行镜像生成 “容器”（独立运行实例），实现 “一次打包，任意环境运行”，轻量、隔离、便携。
