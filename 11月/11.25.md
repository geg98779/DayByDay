## gemini clone dry-code
1. 核心入口文件
src/main.tsx：应用入口点，负责初始化主题、挂载路由和根组件。
src/App.tsx：根组件，配置 Ant Design 主题（明暗模式）、国际化和路由出口（Outlet）。
src/router/index.tsx：路由配置文件，定义了页面路由规则（如登录页、首页、分享会话页等）。
2. 页面组件（src/pages/）
包含应用的主要页面：
Login/index.tsx：登录页面，支持邮箱登录和第三方登录（Google、Microsoft、Apple）。
Home/index.tsx：首页，包含聊天气泡（ChatBubble）和富文本输入框（AIRichInput）。
SharedChat/index.tsx：分享会话页面，展示通过分享链接访问的聊天记录。
Agents/index.tsx：AI 助手列表页，展示可选择的 Agent 并支持跳转对话。
3. 组件（src/components/）
通用 UI 组件和业务组件：
ThemeToggle：主题切换组件（明暗模式切换）。
AIRichInput：聊天输入框组件，支持文本输入和文件处理。
ChatBubble：聊天消息展示组件。
LayoutWithSidebar：带侧边栏的布局组件。
WithPermission：权限控制组件，用于保护需要登录的路由。
4. 状态管理（src/store/）
使用 zustand 管理应用状态：
useUserStore：用户状态（登录状态、token 等）。
useThemeStore：主题状态（明暗模式）。
useLocaleStore：国际化状态（语言切换）。
useChatStore：聊天会话状态（消息列表、当前会话等）。
5. API 接口（src/apis/）
封装后端接口调用：
chat.ts：聊天相关接口（发送消息、文件上传、SSE 连接等）。
user.ts：用户相关接口（登录、注册、验证码等）。
session.ts：会话相关接口（获取分享会话等）。
6. 工具函数与配置
src/utils/request.ts：封装 HTTP 请求工具（基于 axios）。
src/constant/index.ts：常量定义（如接口基础 URL、默认消息等）。
src/locales/：国际化配置（中 / 英文翻译文本）。
7. 样式文件
src/index.css：基础样式，集成 Tailwind CSS 并定义明暗模式基础样式。
src/styles/：其他样式文件（过渡动画、Agent 页面样式等）。
查看方式建议
从入口文件 main.tsx 开始，了解应用初始化流程。
通过 router/index.tsx 梳理页面路由关系。
根据具体功能（如登录、聊天、主题切换）定位到对应页面 / 组件和状态管理文件。
结合 API 目录了解前后端交互逻辑。
按照这个结构，可以系统地浏览 PC 端应用的源码实现。
