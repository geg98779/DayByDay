## 如何封装一个组件
### 明确组件职责
1. 组件的目标: 展示型还是功能型 ui 逻辑
2. 是否具备复用价值：是否在多个场景中出现，是否可以抽象公共逻辑

### 组件名称
根据组件的类型和关键特征进行命名
组件props：
数据型 配置型 事件型 插槽型

### 组件方法
指组件通过useimperativeHandle暴露的父组件可以调用的方法
能用属性和事件的实现组件尽量用属性和事件，实在不行再用方法。react更偏向声明式编程，而组件方法使用命令式编程，少用

### 组件state
注重并且只保留组件内部的状态 可由props或其他state推导出的，使用usememo包裹一下

### 组件样式
1. 不要使用css module，因为使用module，就无法知晓css类名，进行局部样式微调时比较困难
2. 组件最外层的元素样式名称和组件名称保持一致，提供一个前缀

## 组件库如何联调
需要一个用到此组件库的项目 通常在开发阶段 使用npm link来进行联调
npm link： 是一种把包链接到包文件夹的方式，即可以在不发布npm的情况下 调试该模块 并且修改模块后无需npm i 会实时生效

1. 模块和项目在同一目录下： 直接npm link ../module
2. 模块和目录不在同一目录下：
 先去到模块目录，把它link到全局： cd ../npm-link-test npm link
 再去项目目录 cd ../my-project-link npm link test-npm-link

## css样式隔离
1. bem命名方式： 通过命名规则避免样式冲突
2. css-module： 通过编译生成不冲突的选择器类名， 打包工具会将类名编译成带哈希的字符串
3. 预处理器： less sass bem
4. css in js： 天然隔离
