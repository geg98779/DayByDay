## monorepo
monorepo 指单一代码仓库，具体为将多个相关项目或组件存储在单个版本控制存储库中的软件开发方法。传统上，每个项目或组件都有自己独立的代码仓库，但在monorepo架构中，所有项目或组件共享一个共同的代码库。
- 优点：代码共享、一致性和可维护性、跨项目协作、更简化的构建与部署

## 前端工程化
提升开发效率、代码质量、协作能力与可维护性
手段:
1. 项目初始化阶段：标准化启动
用脚手架（比如 Vue CLI、Create React App 或自定义脚手架）快速生成项目骨架，同时集成TS做类型约束、Husky做 Git 提交钩子（比如校验提交信息格式），从启动阶段就统一规范。
2. 编码阶段：保障规范与效率
代码规范：靠 ESLint（语法 + 代码质量校验）、Prettier（自动格式化）、StyleLint（样式规范）实现团队代码风格统一；
模块化设计：通过组件抽象（复用 UI 组件）、Hooks 封装（复用业务逻辑）、目录分层（比如按 “页面 / 组件 / 工具 / 接口” 划分目录），降低代码耦合；
自动化开发：用 Mock 工具模拟接口数据、接口管理工具（比如 Apifox）统一接口文档、环境变量区分开发 / 测试 / 生产环境，提升开发效率。
3. 功能核心层：状态与路由管理
通过状态管理工具（比如轻量的 Zustand 或生态完善的 Redux）管理全局状态，配合路由工具（React Router/Vue Router）做页面路由管控，保障复杂业务的状态与页面流转稳定。
4. 质量保障与交付阶段
测试体系：用 Jest 做单元测试、Playwright 做端到端（E2E）测试，覆盖 “组件逻辑” 到 “全链路流程” 的质量验证；
持续集成：借助 GitHub Actions 实现代码提交后的自动测试、构建，结合 Docker 做环境镜像，保障交付的一致性与效率。
5. 协作与维护阶段
用 Storybook（组件文档 + 可视化调试）、dumi（文档站点生成）沉淀组件 / 业务文档，降低团队协作的沟通成本，也方便后续项目维护。
这套体系的核心是从 “启动 - 编码 - 交付 - 维护” 全流程落地工程化，既提升开发效率，也保障代码质量和协作体验。
